1. Indrodução
Pessoa 1 (0:00 - 0:30): "Olá a todos! Nosso grupo é composto por Eric, Leticia e Lucas. Hoje vamos falar sobre uma linguagem que foi criada com um propósito muito poético: fazer os programadores felizes. Vamos apresentar a linguagem Ruby e mostrar como ela eleva a Orientação a Objetos ao seu nível máximo, onde absolutamente tudo é um objeto."
2. História e contexto
Pessoa 1 (0:30 - 2:00): "Ruby foi criada em 1995 por Yukihiro Matsumoto, conhecido como Matz, no Japão. O contexto de criação era simples: Matz queria uma linguagem que fosse mais poderosa que o Perl e mais orientada a objetos que o Python. A filosofia principal do Ruby é que a linguagem deve parecer natural para os humanos e não para as máquinas. Seu principal campo de aplicação hoje é o desenvolvimento Web, impulsionado pelo famoso framework Ruby on Rails, que é a base de gigantes como GitHub, Shopify e Airbnb."
3. Sintaxe básica
Pessoa 2 (2:00 - 4:00): "Passando para a sintaxe, Ruby é elegante, legível e dinamicamente tipada, o que significa que não precisamos declarar o tipo das variáveis explicitamente. Não usamos ponto e vírgula no final das linhas e a indentação não é obrigatória para o compilador, mas usamos a palavra end para fechar blocos. Aqui na tela, vemos que a leitura do código se aproxima da linguagem natural, reduzindo a complexidade e facilitando a compreensão. Uma das grandes forças da sintaxe são os 'blocos', que nos permitem iterar sobre coleções de forma muito expressiva, como no exemplo 5.times { puts 'Olá' }."
4. Abstração e encapsulamento
Pessoa 2 (4:00 - 6:30): "Entrando nos conceitos de Orientação a Objetos: em Ruby, a abstração é feita através de classes, usando a palavra reservada classe. O método construtor sempre se chama initialize. O encapsulamento é rigoroso com o estado do objeto: as variáveis de instância começam com o símbolo @ (como @saldo) e sempre são privadas por padrão. Você não consegue acessá-las de fora da classe a menos que crie métodos específicos para isso, ou use macros de leitura e escrita como o attr_accessor."
5. Herança e polimorfismo
Pessoa 3 (6:30 - 9:00): "Para a Herança, Ruby utiliza o símbolo de menor <. Diferente de C++, Ruby só permite herança simples, mas resolve o compartilhamento de múltiplos comportamentos usando 'Módulos' (Mixins). Já o Polimorfismo brilha através do conceito de Duck Typing (Tipagem Pato): 'Se anda como um pato e faz quack como um pato, então é um pato'. Em Ruby, o mais importante não é a classe do objeto, mas sim se ele responde ao método esperado, o que caracteriza o polimorfismo baseado em comportamento. Se passarmos um Cachorro ou um Gato para uma função que chama o método falar, o próprio Ruby resolve qual versão executar em tempo de execução."
6. Sobrecarga e programação genérica
Pessoa 3 (9:00 - 11:30): "Aqui está a parte mais interessante de Ruby comparada ao Java ou C++: Ruby não possui sobrecarga de métodos (Overloading) no sentido tradicional. Se você escrever dois métodos com o mesmo nome na mesma classe, o último sobrescreve o primeiro. Como resolvemos isso? Usando parâmetros com valores padrão ou o operador splat (*args), que permite receber um número variável de argumentos num único método. Da mesma forma, Ruby não possui Programação Genérica (Generics). Como a linguagem é dinamicamente tipada, um Array em Ruby já aceita naturalmente qualquer tipo de objeto misturado (números, strings, outras classes), não havendo necessidade de definir um <T> como fazemos em C++."

8. Conclusão
Pessoa 1 (14:00 - 14:30): "Concluindo, Ruby é uma linguagem onde a Orientação a Objetos não é apenas uma funcionalidade adicionada, mas a essência do seu design. Embora lide com sobrecarga e genéricos de maneira não convencional através de tipagem dinâmica e duck typing, ela oferece um ambiente produtivo, com alta expressividade, mas exige maior atenção do desenvolvedor devido à tipagem dinâmica em tempo de execução. Agradecemos a atenção!"
